Modular CSS

Modular CSS means breaking the page up into its component parts. These parts should be reusable in multiple contexts, and they shouldn't directly depend upon one another. You also should be able to concisely summarize its purpose. When a module tries to do more than one thing, you should consider breaking it into smaller modules.

A simple module

Let's create a simple module for brief notification messages. Because each module needs a unique name, you'll call this one "message." You'll give it a bit of color and a border to capture the user's attention

                    
                        <div class="message">
                            Save successful
                            <button class="close" id="close" type="button">close</button>
                        </div>           
                     
                
                                        
                        .message {
                            padding: 0.8em 1.2em;
                            border-radius: 0.2em;
                            border: 1px solid #265559;
                            color: #265559;
                            background-color: #e0f0f2;
                            position: relative;
                        }
Save successful

It's important that the selector for this module consists only of the single class name. Nothing else in the selector restricts these styles to a certain place on the page.In contrast, using a selector like #sidebar .message would mean this module could only be used inside a #sidebar element. Without this restriction, the module is reusable in any context.

Variations of a module

Consistency is good, but sometimes you’ll want to intentionally deviate from it. Our Message module is nice, but we might need it to look different under certain circumstances. For instance, if you need to display an error message, perhaps it should be colored red rather than teal. Or, perhaps you want to distinguish between messages that are purely informational and those that indicate a successful action, such as saving. You do this by defining modifiers.

                    
                        <div class="message message--success">
                            Save successful
                            <button class="close" id="close" type="button">close</button>
                        </div>           
                     
                
                                            
                            .message {
                                padding: 0.8em 1.2em;
                                border-radius: 0.2em;
                                border: 1px solid #265559;
                                color: #265559;
                                background-color: #e0f0f2;
                                position: relative;
                            }
    
                            .message--success {
                                color: #2f5926;
                                border-color: #2f5926;
                                background-color: #cfe8c9;
                            }
Save successful

Button Module Variants

Let's create another module with some variants. You’ll make a Button module with variants for large and small sizes, as well as color options. This way, you can use color to add visual meaning to the buttons. Green will indicate a positive action, such as saving or submitting a form. Red will indicate a warning to help prevent the user from accidentally clicking a cancel button.

                    
                        <button type="button" class="button button--large">Read more</button>
                        <button type="button" class="button button--success">Save</button>
                        <button type="button" class="button button--danger button—small">Cancel</button>
           
                     
                
                                            
                            .button {
                                padding: 0.5em 0.8em;
                                border: 1px solid #265559;
                                border-radius: 0.2em;
                                background-color: transparent;
                                font-size: 1rem;
                            }
                            
                            .button--success {
                                border-color: #cfe8c9;
                                color: #fff;
                                background-color: #2f5926;
                            }
                            
                            .button--danger {
                                border-color: #e8c9c9;
                                color: #fff;
                                background-color: #a92323;
                            
                            }
                            
                            .button--small {
                                font-size: 0.8rem;
                            }
                            
                            .button--large {
                                font-size: 1.2rem;
                            }

Modules with multiple elements

The two modules youve built so far—message and button—are nice and simple; they consist of only one element. But many modules you'll build will need more elements. You can't build a dropdown menu or a modal with only one element. Let's build a more complex module

Strength

Strength training is an important part of injury prevention. Focus on your core— especially your abs and glutes.

This module consists of four elements: it has a div container that includes an image and a body, and inside the body is the title. As with other modules, you'll give the main container the class name media to match the name of the module. For the image and the body, you'll use the class names media__image and media__body. These begin with the module name, followed by a double-underscore, then the name of the sub-element. (This is another convention from BEM methodology.) As with double-hyphen modifiers, the class name tells you at a glance what role the element plays and what module it belongs to.

                        
                    <div class="media">
                        <img class="media__image" src="runner.png">
                        <div class="media__body">
                            <h4>Strength</h4>
                            <p>
                                Strength training is an important part of
                                injury prevention. Focus on your core—
                                especially your abs and glutes.
                            </p>
                        </div>
                    </div>           
                         
                    
                                                
                                .media {
                                    padding: 1.5em;
                                    background-color: #eee;
                                    border-radius: 0.5em;
                                }
                                
                                .media::after {
                                    content: "";
                                    display: block;
                                    clear: both;
                                }
                                
                                .media__image {
                                    float: left;
                                    margin-right: 1.5em;
                                }
                                
                                .media__body {
                                    overflow: auto;
                                    margin-top: 0;
                                }
                                
                                .media__body>h4 {
                                    margin-top: 0;
                                }

Modules composed into larger structures

Let's build the dropdown menu with two different modules. The first, which I'll call dropdown, will have a button responsible for controlling the visibility of a container. The second module, which I'll call menu, will be the list of stacked links. You'll then compose the full interface by placing an instance of the menu module inside the container of a Dropdown module.

  • Home
  • Coffees
  • Brewers
  • Specials
  • About us
                    
                    <div class="dropdown">
                        <button type="button" class="dropdown__toggle">Main Menu</button>
                        <div class="dropdown__drawer">
                            <ul class="menu">
                                <li class="menu__item" ><a href="#">Home</a></li>
                                <li class="menu__item" ><a href="#">Coffees</a></li>
                                <li class="menu__item" ><a href="#">Brewers</a></li>
                                <li class="menu__item" ><a href="#">Specials</a></li>
                                <li class="menu__item" ><a href="#">About us</a></li>
                            </ul>
                        </div>
                    </div>

I've used the double-underscore notation here to indicate that toggle and drawer are sub-elements of the Dropdown module. Clicking the toggle will reveal or hide the drawer. The styles for the dropdown are shown in the next listing. Add this to your stylesheet. This gives you a functioning dropdown, though the menu inside it remains unstyled

                      
                        .dropdown {
                            position: relative;
                            display: inline-block;
                        }
                        
                        .dropdown__toggle {
                            padding: 0.5em 2em 0.5em 1.5em;
                            border: 1px solid var(--default-bd-color);
                            font-size: 1rem;
                            background-color: var(--default-bg-color);
                        }
                        
                        .dropdown__toggle::after {
                            content: "";
                            position: absolute;
                            right: 1em;
                            top: 1em;
                            border: 0.3em solid;
                            border-color: black transparent transparent;
                        }
                        
                        .dropdown__drawer {
                            display: none;
                            position: absolute;
                            left: 0;
                            top: 1.0em;
                            width: 100%;
                            background-color: var(--default-bg-color);
                        }

With the Dropdown module working, you can now focus your attention on the Menu module. You don't need to worry any longer about the opening and closing of the dropdown, as that's already taken care of by the Dropdown module. The Menu module will apply the look and feel you need to the list of links. 

                                                      
    
                            .menu {
                                margin: 0;
                                padding-left: 0;
                                list-style: none;
                                border: 1px solid var(--default-bd-color);
                            }
                            
                            .menu__item+.menu__item {
                                border-top: 1px solid var(--default-bd-color);
                                margin-top: 0px;
                            }
                            
                            .menu__item>a {
                                display: block;
                                padding: 0.5em 1.5em;
                                text-decoration: none;
                                color: #369;
                                background-color: var(--default-bg-color);    
                            }
                            
                            .menu__item>a:hover {
                                background-color: var(--default-sc-color);
                            }

Sumary

  • Break your CSS up into small, reusable modules.
  • Never write styles that reach into another module and change its appearance.
  • Use variant classes to provide multiple versions of the same module.
  • Divide large constructs into smaller modules; build your pages by piecing together a number of modules.
  • Group all rules for a module together in your stylesheet.
  • Use a naming convention such as double-hyphens and double-underscores to make your modules' structure easier to understand at a glance.
